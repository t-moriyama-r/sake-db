# フロントエンド開発ルール
## ディレクトリ構成
基本的に触るディレクトリを太字にしている。
| ディレクトリ | 説明                              |
|------------------|-----------------------------------|
| .devcontainer          | コンテナ設定ファイル       |
| .vscode            | なんであるのか覚えてない。VSコード用の設定                  |
| dist               | 自動生成されたやつっぽい。|
| public            | 静的ファイル置き場？ファビコンしかない。    |
|**src**|ここをメインに触る。(詳細は下記)|

| src/ | 説明                              |
|------------------|-----------------------------------|
| assets          | 静的ファイル       |
| **components**           | コンポーネント                  |
| const               | 定数の定義|
|funcs|共通ロジックの定義。ヘルパ関数とか。|
|**graphQL**|スキーマ定義。interfaceが定義されたりするのは初期の残骸。(詳細は後述)|
|lib|ライブラリ関連のファイル|
|plugins|プラグインとして使うロジック|
|**router**|ルーティングの定義|
|stores|ストアの定義|
|styles|スタイルシート|
|types|汎用インターフェースの定義|

### コンポーネント分類
| ディレクトリ | 説明                              |
|------------------|-----------------------------------|
|**views**|基本的にコンポーネントはここで定義される|
| components/layouts          | レイアウトを定義する。children相当を必ず持つ。       |
| components/blocks          | partsを組み合わせて作るような共通コンポーネント                  |
| components/parts               | コンポーネントの最小単位。他のコンポーネントをimportしない。|
| components/slots            | スロットコンポーネントを定義する。    |

viewsとcomponentsという命名が気に入らないので、そのうちどっかでリファクタしたい気持ちはある。

## GraphQLの使い方
graph配下のクエリを弄った時、型情報を更新する必要がある。<br/>
`npm run codegen`<br/>
<br/>
バックエンド側でスキーマを弄った際は、フェッチしないと警告が出るはず。
`npm run fetch-schema`<br/>
※サーバーを起動していないと通信できない
